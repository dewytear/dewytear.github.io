# 다국어(i18n) 작업 절차서

이 위키의 번역·언어 추가 작업이 따라야 하는 **단일 기준 체크리스트**입니다. 어떤 언어(en/zh/ja/…)든 아래 순서를 그대로 따르면 누락이 없습니다.

## 구조 한눈에

| 조각 | 위치 | 폴백 |
|---|---|---|
| 문서 본문 | `docs/<lang>/<path>` (한국어 원본과 **같은 상대 경로** — `list` 노드의 `path`, 없으면 flat `<문서명>`) | 파일 없으면 `docs/ko/` |
| 화면(UI) 문구 | `index.html`의 `STRINGS.<lang>` (`STR()`/`STRF()` 사용) | 키 없으면 `STRINGS.ko` |
| 내비 라벨·태그 | `list` 노드의 `label_<lang>` / `tags_<lang>` (폴더는 `title_<lang>`) | 없으면 한국어 `label`/`tags` |
| 지식 인덱스 | `tools/doc-entries.<lang>.json` (오버레이) → `data/knowledge-index.<lang>.json` | 파일 없으면 ko 인덱스 |
| 지도 문서 | `docs/<lang>/ai-map` (km-* 하이드레이션 id 유지) | 없으면 ko ai-map |
| 클러스터 표시명 | `tools/build_index.py`의 `CLUSTER_LABELS_BY_LANG[<lang>]` | 없으면 ko 라벨 |

언어 선택: 설정 → 기본 설정 → 언어 (`s.lang`, 사이트 기본값 `config.json defaults.lang`). 변경 저장 시 새로고침으로 일괄 적용.

## 핵심 원칙 (반드시 지킬 것)

1. **본문 파생 데이터는 본문 번역과 한 몸.** 문서의 `label_<lang>`·`tags_<lang>`·`doc-entries.<lang>.json`의 title/summary 오버레이는 **그 문서의 본문을 번역하는 PR에서 함께** 갱신한다. 본문보다 먼저 하지 않는다(제목↔본문 불일치 방지). Introduction 6편이 선례.
2. **UI 문구(STRINGS)는 본문과 무관하게 선행 가능.** 새 UI 문구를 코드에 추가할 때는 반드시 `STR()`/`STRF()` 키로 넣고 `STRINGS.ko`와 준비된 모든 언어 사전에 같이 등록한다 — 하드코딩 한글 리터럴 금지.
3. **개념(concepts)은 언어 공유 키.** 오버레이는 title·summary만 바꾸고 concepts는 ko 것을 그대로 상속 — 연관 관계·stats·그래프 구조가 언어와 무관하게 동일해야 한다.
4. **내부 링크는 canonical(`#!문서명`) 유지.** 번역 파일 안에서도 링크를 언어별로 바꾸지 않는다(라우팅이 언어를 해석).
5. **본문 번역 규칙**: 내용 추가·수정·삭제 금지, HTML 구조·클래스·SVG 지오메트리 보존, SVG 텍스트 라벨은 번역(전문용어·코드는 원형), aria-label도 번역.

## 언어 하나를 추가/확장하는 순서

1. `docs/<lang>/`에 번역 파일 배치 — **한국어와 같은 상대 경로**(`list` 노드의 `path`)에 둔다 (부분 가능 — 나머지는 ko 폴백)
2. `index.html`: `LANGS_READY`에 `<lang>` 추가, `STRINGS.<lang>` 사전 채우기(키 목록은 `STRINGS.ko` 기준 — 빠진 키는 ko 폴백)
3. 번역한 각 문서마다(원칙 1): `list`에 `label_<lang>`(+필요시 `tags_<lang>`), `tools/doc-entries.<lang>.json`에 `{name, title, summary}` 오버레이 추가
4. `tools/build_index.py`의 `CLUSTER_LABELS_BY_LANG[<lang>]` 등록(새 클러스터가 생기면 **모든 언어**에 추가) → `python3 tools/build_index.py` 실행(`--check` 0)
5. `docs/<lang>/ai-map` 작성 — 본문 번역 + km-* id·정적 폴백 수치 유지, `data-topics` 키는 그 언어의 클러스터 라벨과 일치
6. 설정 select 표기: 콘텐츠가 충분해지면 `STRINGS`의 `lang<Xx>` 값에서 "준비 중/coming soon" 제거
7. Work Log 폴더 제목("07월"·"05일") 등 구조 라벨은 해당 언어 콘텐츠가 생길 때 `title_<lang>`으로
