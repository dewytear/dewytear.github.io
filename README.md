# Aaron's Claude Wiki

Claude Code · 플러그인 · 하네스 · LLM Wiki · 세컨드 브레인을 다루는 한글 위키입니다.
`fetch()` API로 문서를 동적으로 불러오는 싱글 페이지(SPA) 방식으로 동작하며,
GitHub Pages로 배포됩니다: https://dewytear.github.io

## 주요 기능

### 콘텐츠 · 탐색
- **동적 콘텐츠 로딩** — `index.html`이 `fetch()`로 문서 조각(90여 편)을 불러와 렌더링합니다.
- **트리형 네비게이션** — `list`(JSON) 트리로 사이드바를 자동 생성. 폴더 옆 ⧉ 아이콘으로 그 폴더의 문서를 **한 페이지로 모아보기**.
- **URL 해시 라우팅** — `#!install`(문서), `#!tags`(태그), `#!folder:…`(모아보기), `#!cosmos`(코스모스), `#!search`(검색), `#!settings`(설정).
- **통합 검색** — 제목·태그·본문 전문에 더해 지식 인덱스의 **개념까지 가중 랭킹**으로 검색(개념 매칭 문서가 상단, 관련 개념 피벗 칩 제공). 배경엔 호흡 애니메이션과 미니게임.
- **태그 인덱스** — 사전식 초성/알파벳 그룹, 최다 사용 태그 밴드, 즉시 필터.

### AI 지식 관리 시스템
- **지식 인덱스**(`knowledge-index.json`) — AI가 **AI 대분류 아래의 강의 문서**를 전부 읽어 문서별 요약·핵심 개념·연관 문서를 기록. `tools/build_index.py`로 **결정적 재생성**(`--check` 드리프트 검사).
- **❖ 지식 지도**(`#!ai-map`) — 클러스터·허브·브리지 개념을 집계한 지도. 표의 수치는 인덱스의 `stats`에서 **실시간 렌더**(드리프트 원천 차단).
- **✦ 지식 코스모스**(`#!cosmos`) — 연관 관계를 **3차원 별자리**로: 분류별 은하(이름표), 문서=별, 연관=연결선. 드래그 회전·휠 확대·별 클릭 시 문서 이동.
- **연관 문서 추천** — 각 문서 하단에 의미 기반 추천 카드(공유 개념 칩 클릭 → 검색 피벗).
- **AI 큐레이터** — `tools/curator.md` 절차대로 변경 감지 → 재인덱싱 → 검토용 PR 제안(자동 머지 없음).

### 놀이
- **미니게임 4종** — 통합검색 배경에서 벽돌깨기(검색창=드래그 패들, 반사각 조절) · 개념 별똥별 받기(받으면 그 개념으로 검색) · 퐁 랠리 · 종이비행기 글라이딩. 화면 하단 **게임 독**에서 방문자가 직접 선택(브라우저별 기억), 설정에선 기본 게임 지정.

### 디자인 · 설정
- **클로드식 셸 레이아웃** — 데스크톱에서 틴트 배경이 본문 카드를 감싸고, 좌측 메뉴는 부드러운 슬라이드 개폐(기본 = 접힘, 선택 기억). 메뉴가 열릴 때만 카드가 은은하게 나타나는 모핑.
- **낮/밤 테마** — 라이트(오렌지 포인트)·다크(teal 포인트) 전환.
- **설정 패널**(암호 보호, 기본/디자인 탭) — 타이틀·태그라인·사진 문구·배경음악(YouTube)·목록 구분선 모양/두께·최근/연관 문서 표시·대분류 노출·기본 미니게임·**낮/밤 테마 포인트 색상**(컬러피커, 즉시 미리보기). 하단 **진단 패널**로 이 브라우저의 항목별 적용 값과 출처(개인/사이트/앱 기본)를 확인·복사.
- **사이트 기본값 vs 개인 설정** — `config.json`의 `defaults` 블록(테마 색·기본 게임·구분선·표시 옵션 등)이 **모든 방문자**의 시작값. 설정 패널에서 저장한 값은 **그 브라우저에서만** 우선 적용되고, 기본값과 같은 값은 저장하지 않아 이후 기본값 변경이 계속 전달됩니다.
- **배경음악** — YouTube IFrame 재생, 차단 감지 시 안내와 폴백 링크.
- **반응형** — 모바일은 드로어 메뉴 + 단일 컬럼.

## 파일 구조

| 파일 | 설명 |
|------|------|
| `index.html` | 메인 SPA — 라우팅·검색·지식 인덱스 로딩·미니게임·코스모스 |
| `style.css` | 테마 토큰·셸 레이아웃·반응형 스타일 |
| `colors.js` | 낮/밤 테마 토글 로직 |
| `list` | 네비게이션 트리 정의 (JSON) |
| `config.json` | 서버 전역 설정 — 타이틀·프로필 사진 + `defaults`(모든 방문자의 설정 기본값) |
| `knowledge-index.json` | AI 지식 인덱스(요약·개념·연관·stats) — 직접 편집 금지 |
| `ai-map` | ❖ 지식 지도 문서 |
| `tools/build_index.py` | 인덱스 결정적 빌드(`--check` 검증) |
| `tools/doc-entries.json` | 인덱스 원료(문서별 AI 작성 엔트리) |
| `tools/curator.md` | AI 큐레이터 절차서 |
| `welcome`·`ccb-*`·`skl-*`·`hns-*`·`hnp-*`·`llmw-*`·`sb-*`·`omc-*` | 강의 문서 조각(HTML) |
| `wl-*` | Work Log(개발 일지 — 태그·최근 문서 없음) |

## 문서 추가하기

1. 새 문서 파일을 만듭니다 (예: `settings`). 내용은 `<h2>...</h2><p>...</p>` 형태의 HTML 조각으로 작성합니다.
2. `list`(JSON) 트리에 노드를 추가합니다. 링크 노드는 `{ "name": "settings", "label": "Settings", "tags": ["config", "settings"] }`, 섹션은 `{ "title": "...", "children": [ ... ] }` 형태이며 `children`으로 얼마든지 중첩할 수 있습니다. `tags`는 선택 사항이며, 지정하면 문서 하단과 태그 인덱스에 자동으로 반영됩니다.
3. **작성 AI·모델 표기** — AI가 문서를 추가하거나 대폭 다시 쓸 때는 노드에 `"model": "<작성 세션의 모델명>"`을 함께 기록합니다(제목 우측 배지로 표시). 이 값은 **그 문서를 실제로 쓴 세션의 정보**를 따르며, 없으면 기본값(`DOC_MODEL`)이 표시됩니다.
4. 커밋하면 사이드바 트리에 자동으로 나타납니다.

## 프로필 이미지 바꾸기

좌측 상단 이미지는 `profile.svg`(플레이스홀더)입니다. 원하는 이미지로 바꾸려면
`profile.svg`를 교체하거나, `index.html`의 `#profile-img` 태그 `src`를 원하는 파일
경로(예: `profile.jpg`)로 수정하면 됩니다.

## 로컬 실행

`fetch()`는 `file://` 프로토콜에서 동작하지 않으므로 로컬 서버가 필요합니다.

```bash
# Python 3
python3 -m http.server 8000
```

브라우저에서 http://localhost:8000 접속.
